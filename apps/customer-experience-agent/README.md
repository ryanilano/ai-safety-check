# Customer Experience Intelligence Agent

Turns a customer ID into a personalized engagement brief — product recommendations, targeted discount offers, and a CRM action trigger — all without writing SQL.

The agent autonomously discovers the TheLook E-Commerce schema via the CRAFT semantic platform, runs natural-language queries across purchase history, behavioral events, and product catalog, then feeds all evidence to Gemini to produce a ready-to-act markdown report.

---

## How It Works

```
python main.py 12345
        │
        ▼
┌─────────────────────┐
│ Node 1              │  get_schema → discover tables
│ DISCOVER PROFILE    │  generate_sql + execute_query → customer demographics
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ Node 2              │  Q1: purchase history (ORDERS × ORDER_ITEMS × PRODUCTS)
│ ANALYZE BEHAVIOR    │  Q2: category spend breakdown
└────────┬────────────┘  Q3: event type distribution (EVENTS)
         ▼
┌─────────────────────┐
│ Node 3              │  Q1: top products in preferred categories not yet bought
│ RECOMMEND PRODUCTS  │  Q2: high-margin products for targeted discount offers
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ Node 4              │  Chart 1: category spend bar chart
│ VISUALIZE           │  Chart 2: recommended products price bar chart
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ Node 5              │  Gemini synthesis → personalized engagement brief
│ COMPOSE ENGAGEMENT  │  (recommendations · offers · CRM trigger · risk)
└────────┬────────────┘
         ▼
  runs/customer_12345_*/
    engagement_brief.md
```

## Setup

### 1. Clone / navigate to the project

```bash
cd apps/customer-experience-agent
```

### 2. Install dependencies

```bash
uv sync
```

### 3. Configure environment

Copy the template and fill in the four required values:

```bash
cp .env.template .env
```

Open `.env` and set:

```bash
# CRAFT MCP endpoint
MCP_URL=https://your-runtime-host/mcp
PROJECT_ID=your-project-id-here

# Keycloak base URL (no /realms suffix)
KEYCLOAK_URL=https://your-keycloak-host

# TheLook data connection (from your Craft project settings)
DATASOURCE_ID=your-datasource-uuid
CONNECTION_NAME=your-connection-name
SCHEMA_NAME=THELOOK_ECOMMERCE
SCHEMA_FQN=your-connection.DATABASE.SCHEMA

# Gemini API key — get from https://aistudio.google.com
GEMINI_API_KEY=your_key_here

# Leave this blank for now — filled in step 4
KEYCLOAK_REFRESH_TOKEN=
```

### 4. Obtain a Keycloak refresh token (one-time)

This opens a browser tab for Google SSO login. Run it once; the token is long-lived (days to weeks).

```bash
uv run python scripts/get_token.py
```

The script prints:

```
KEYCLOAK_REFRESH_TOKEN=eyJhbGci...
```

Paste that value into your `.env` file.

```bash
# Run for a specific customer ID
```

The agent prints live progress for each node and the final engagement brief to the terminal.

## MCP Tools Used

| Tool | Node | Purpose |
|------|------|---------|
| `get_schema` | 1 | Discover TheLook table list |
| `generate_sql` | 1, 2, 3 | Natural language → SQL (Text2SQL) |
| `execute_query` | 1, 2, 3 | Run SQL, get artifact reference |
| `get_result_page` | 1, 2, 3 | Fetch rows from artifact (called internally by `execute_query`) |
| `generate_plotly_chart` | 4 | Build chart specs from row data |

~15 MCP tool calls per run.

## Retry Behaviour

| Layer | Trigger | Action |
|-------|---------|--------|
| Graph | No behavioral data after Node 2, retries < 2 | Backtrack to Node 1 |
| Client | `talk2data_unreachable` from `generate_sql` | Retry up to 2× (2 s, 4 s backoff) |
| Transport | `httpx.TimeoutException` / `ConnectError` | Retry up to 4× (1–8 s backoff, tenacity) |
| Auth | 401 on any MCP call | Refresh token, retry once |

## Troubleshooting

**`Required environment variable 'KEYCLOAK_REFRESH_TOKEN' is not set`**
→ Run `uv run python scripts/get_token.py` and paste the token into `.env`.

**`Keycloak token refresh failed (400)`**
→ Refresh token has expired. Re-run `scripts/get_token.py` to get a new one.

**`get_schema unavailable (likely insufficient metadata permissions)`**
→ This is a non-fatal warning. The agent continues using a hardcoded table list. No action required — it does not affect data queries.

**`generate_sql failed [...]` / `talk2data_error`**
→ Talk2Data could not generate SQL (often because schema metadata is unavailable). The agent automatically falls back to pre-written SQL for every query and continues. Check `runs/.../agent.log` for the full error. If all queries fail, verify your `MCP_URL` and `KEYCLOAK_REFRESH_TOKEN` point to the correct environment.

**`execute_query failed`**
→ The agent reached the database but the SQL failed to execute. Check `runs/.../agent.log` for the raw error. Common causes: wrong table reference format for this environment, or the customer ID column type mismatch. The fallback SQL uses `THELOOK_ECOMMERCE.USERS` — confirm the schema prefix matches your data connection.

**`No customer found with id=12345`**
→ The customer ID does not exist in the TheLook database. Try a different integer ID (e.g. `python main.py 99001`).

**Charts not rendering**
→ Confirm `plotly` is installed (`uv run python -c "import plotly"`). Use `pio.from_json(...)` not `json.load(...)` directly.
